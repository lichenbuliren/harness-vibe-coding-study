# Checklists

This directory will store repeatable completion and quality checklists.

Checklists should be concrete enough for agents to run before claiming work is
complete.

## Current Checklists

- `mobile-ui-state-feedback.md`: checks that mobile UI actions expose disabled
  and ready states clearly and testably.
- `mobile-interaction-performance.md`: checks that mobile interactions keep
  state local, avoid avoidable rerenders, and verify the user-facing access
  path.
