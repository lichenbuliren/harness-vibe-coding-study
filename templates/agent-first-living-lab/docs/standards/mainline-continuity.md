# Mainline Continuity

Long agent conversations can drift. Useful local work should not erase the
larger mission.

## Standard

After a meaningful subtask, correction loop, side exploration, or stage-level
artifact, return to the project mainline before closing or branching again.

Use this check:

```text
larger goal -> local work completed -> current project position ->
next mainline step
```

## Expected Output

The agent should make the continuity state visible in the final response or in
durable docs when the phase or learning changed.
